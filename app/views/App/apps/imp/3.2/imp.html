IMP 3.x is no longer actively developed

No further official releases of the IMP 3.x branch will be created outside of critical security releases. The Horde Project strongly urges sites still running IMP 3.x to upgrade to 4.x as soon as possible.

IMP 3.2.8 (final) is now available. This is a security fix release on top of 3.2.7. Features that have been added since 3.1 include several workarounds for buggy browser behaviour when downloading attachments, warning popups in the compose window before a user session times out, support for folder names in other character sets than iso-8859-1, virtual filesystem support for attachments that helps deploying IMP on server farms, and many bug fixes, plus all the features found in IMP 3.1 which make 3.x so much better than IMP 2.2.

IMP 3.2.8 requires PHP 4.1.0 or higher, and will operate with versions 2.0 to 2.2.9 of the Horde Application Framework.

Important note: IMP 3.2.8 does not require PHPLIB, and the two may in fact conflict if they overlap. If you are still running IMP 2.2, you can configure PHPLIB to be active only for the IMP 2.2/Horde 1.2 part of your webroot.
