These are instructions to upgrade from earlier Whups versions. Please backup your existing data before running any of the steps described below. You can't use the updated data with your old Whups version anymore.

After updating to a newer Whups version, you always need to update configurations and database schemes. Log in as an administrator, go to Administration => Configuration and update anything that's highlighted as outdated.

1   Upgrading Whups from 1.0 to 1.0.1

1.1   SQL Backends

A field to store whether a version is still active has been added to the default SQL table layout.

Execute the provided SQL script to update your data to the new Whups version, e.g.:

mysql --user=root --password=<MySQL-root-password>  <db name> < scripts/upgrades/1.0_to_1.0.1.sql
