IMP 3.1 is obsolete. IMP 3.1 has now been superceded by IMP 3.2.

Features that have been added since 3.0 include a new address book interface popup during compose, sorting by threads, improved/expanded preference options and config files, folder navigator support for other folder hierarchies, and many bug fixes, plus all the features found in IMP 3.0 which make 3.x so much better than IMP 2.2.

IMP 3.1 requires PHP 4.1.0 or higher, and will operate with either version 2.0 or 2.1 of the Horde Application Framework.

Important note: IMP 3.1 does not require PHPLIB, and the two may in fact conflict if they overlap. If you are still running IMP 2.2, you can configure PHPLIB to be active only for the IMP 2.2/Horde 1.2 part of your webroot.
