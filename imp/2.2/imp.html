Security Information

  • 2003-01-28: IMP 2.2.x contains a SQL injection vulnerability, which can be used by an attacker to execute SQL statements with the privileges of the Horde database user, by simply manipulating Horde URLs.

IMP 2.2 is no longer actively developed

No further official releases of the IMP 2.2.x branch will be created outside of critical security releases. The Horde Project strongly urges sites still running IMP 2.2 to upgrade to 4.x as soon as possible.

IMP 2.2 is the previous stable branch, now obsolete. It used PHPLIB for session management, and worked with both PHP3 and PHP4. Version 2.2 is no longer actively developed.
