Throwing stuff out: when do you get rid of something?

  • When it causes problems: PHPLIB is a huge setup hurdle with IMP 2.2. Using PHP4 sessions vastly simplifies setup.
  • When it isn't being maintained: IMP 2.2's setup program is practically useless now because it's out of date.
  • When it needs to be radically rewritten: IMP's address book code was ugly and hackish. To make sure we rewrote it, I nuked it.
